# 企业年度经营分析项目指导

## 一、项目概述

本项目模拟真实的企业数据分析场景，你需要扮演数据分析师的角色，完成智云科技2024年度经营分析。整个项目分为五个阶段，预计耗时 4-6 小时。

## 二、项目阶段

### 阶段一：数据探索（30分钟）

**目标**：了解数据结构和质量

**操作步骤**：
1. 打开四个 Excel 数据文件
2. 检查每张表的列名、数据类型、记录数
3. 识别数据质量问题：
   - 是否有空值？
   - 日期格式是否统一？
   - 数值是否有异常？
   - 是否有重复记录？

**检查清单**：
- [ ] 财务数据：12个月 × 5个科目 = 60行
- [ ] 销售数据：约100条订单记录
- [ ] 人力资源数据：约80名员工
- [ ] 库存数据：约20种产品

### 阶段二：数据清洗与整合（60分钟）

**目标**：使用 Power Query 将多表整合为干净的数据模型

**操作步骤**：

#### 2.1 导入数据
1. 新建 Excel 文件：`company_analysis.xlsx`
2. 「数据」→「获取数据」→「自文件」→「自 Excel 工作簿」
3. 依次导入四个数据文件
4. 对每个查询进行基础清洗：
   - 检查数据类型
   - 删除空行
   - 统一日期格式

#### 2.2 创建维度表

**产品维度表**：
1. 从库存数据中提取产品ID、产品名称、类别
2. 删除重复项
3. 重命名为「产品维度」

**客户维度表**：
1. 从销售数据中提取客户ID
2. 使用 Power Query 添加客户名称（模拟数据）
3. 重命名为「客户维度」

**日期维度表**：
1. 使用 DAX 创建日期表
2. 或从销售数据中提取唯一日期

#### 2.3 建立事实表

**财务事实表**：
- 直接使用清洗后的财务数据

**销售事实表**：
- 直接使用清洗后的销售数据
- 添加计算列：销售额 = 数量 × 单价

**人力资源事实表**：
- 直接使用清洗后的人力资源数据

### 阶段三：数据建模（45分钟）

**目标**：使用 Power Pivot 建立星型模型

**操作步骤**：

#### 3.1 加载到数据模型
1. 将所有清洗后的查询「关闭并上载到」→「仅创建连接」→「将此数据添加到数据模型」

#### 3.2 建立关系

在 Power Pivot 关系图中建立以下关系：

| 主表（一端） | 从表（多端） | 关联字段 |
|-------------|------------|---------|
| 产品维度 | 销售事实 | 产品ID |
| 客户维度 | 销售事实 | 客户ID |
| 日期维度 | 销售事实 | 日期 |
| 日期维度 | 财务事实 | 月份 |

#### 3.3 创建计算列

**销售事实表**：
```dax
销售额 = 销售事实[数量] * 销售事实[单价]
```

**人力资源表**：
```dax
工龄 = DATEDIFF(人力资源[入职日期], TODAY(), YEAR)
```

### 阶段四：DAX 度量值（60分钟）

**目标**：编写核心度量值，支撑分析需求

#### 4.1 财务度量值

```dax
总收入 := SUM(财务事实[收入金额])
总成本 := SUM(财务事实[成本金额])
总利润 := [总收入] - [总成本]
利润率 := DIVIDE([总利润], [总收入])
成本率 := DIVIDE([总成本], [总收入])
```

#### 4.2 销售度量值

```dax
总销售额 := SUM(销售事实[销售额])
总销量 := SUM(销售事实[数量])
平均客单价 := DIVIDE([总销售额], DISTINCTCOUNT(销售事实[订单ID]))
```

#### 4.3 人力资源度量值

```dax
员工总数 := COUNTROWS(人力资源)
在职员工数 := CALCULATE(COUNTROWS(人力资源), 人力资源[离职状态] = "在职")
离职率 := DIVIDE(CALCULATE(COUNTROWS(人力资源), 人力资源[离职状态] = "离职"), [员工总数])
平均薪资 := AVERAGE(人力资源[基本工资])
```

#### 4.4 库存度量值

```dax
总库存金额 := SUM(库存[库存金额])
平均库存周转率 := AVERAGE(DIVIDE(库存[出库数量], (库存[期初库存] + 库存[期末库存]) / 2))
```

#### 4.5 综合分析度量值

```dax
人均产值 := DIVIDE([总销售额], [在职员工数])
人均利润 := DIVIDE([总利润], [在职员工数])
```

**目标**：基于数据分析结果，撰写分析结论和建议

**报告结构**：
1. **执行摘要**：关键发现和核心建议
2. **财务分析**：收入、成本、利润分析
3. **销售分析**：区域、产品、客户分析
4. **人力资源分析**：人员结构、效能分析
5. **库存分析**：周转、滞销分析
6. **风险提示**：发现的问题和潜在风险
7. **行动计划**：具体的改进建议

## 三、常见问题

### Q1: 数据透视表刷新失败？
A: 检查 Power Query 查询是否正确加载到数据模型，确认关系已建立。

### Q2: DAX 度量值返回错误？
A: 检查公式语法，确认引用的列名和表名正确，注意上下文转换。

### Q3: 仪表板性能慢？
A: 减少数据透视表数量，使用切片器而非多个筛选器，考虑使用 Power BI。

### Q4: 如何保护仪表板？
A: 「审阅」→「保护工作表」，勾选允许使用切片器，取消勾选编辑单元格。
