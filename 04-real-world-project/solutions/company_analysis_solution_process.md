# 企业年度经营分析项目 - 完整解决方案

## 项目概述

本项目是智云科技2024年度经营分析的完整解决方案。以下步骤详细记录了从数据清洗到仪表板构建的全过程。

## 阶段一：数据探索与理解

### Step 1: 检查数据结构

**操作**：
1. 打开四个数据文件：`finance_data.xlsx`、`sales_data.xlsx`、`hr_data.xlsx`、`inventory_data.xlsx`
2. 查看每张表的记录数和字段

**结果**：
| 数据表 | 记录数 | 关键字段 |
|--------|--------|---------|
| 财务数据 | 108行 | 月份、科目类型、科目名称、金额 |
| 销售数据 | 120行 | 订单ID、日期、产品ID、客户ID、区域、数量、单价 |
| 人力资源 | 80行 | 员工ID、姓名、部门、职位、入职日期、基本工资、绩效评分、离职状态 |
| 库存数据 | 25行 | 产品ID、产品名称、类别、期初库存、入库数量、出库数量、期末库存、库存金额 |

**发现的数据特征**：
- 财务数据已按收入和成本分类，便于汇总
- 销售数据需要计算销售额（数量×单价）
- 人力资源数据包含在职和离职员工
- 库存数据可直接计算周转率

## 阶段二：数据清洗与整合（Power Query）

### Step 2: 创建主分析文件

**操作**：
1. 新建 Excel 文件，命名为 `company_analysis.xlsx`
2. 「数据」→「获取数据」→「自文件」→「自 Excel 工作簿」
3. 依次导入四个数据文件

### Step 3: 清洗财务数据

**操作**：
1. 选择财务数据查询
2. 检查数据类型：
   - 月份：文本
   - 科目类型：文本
   - 科目名称：文本
   - 金额：整数
3. 添加「月份序号」列，用于排序：
   - 「添加列」→「自定义列」
   - 公式：`Text.Replace([月份], "月", "")`
   - 转换为整数类型

### Step 4: 清洗销售数据

**操作**：
1. 选择销售数据查询
2. 检查日期格式，确保为日期类型
3. 添加「月份」列：
   - 公式：`Date.ToText([日期], "M月")`
4. 添加「销售额」计算列：
   - 公式：`[数量] * [单价]`
5. 添加「季度」列：
   - 公式：`Date.QuarterOfYear([日期])`

### Step 5: 清洗人力资源数据

**操作**：
1. 选择人力资源查询
2. 确保入职日期为日期类型
3. 添加「工龄」计算列：
   - 公式：`Date.Year(DateTime.LocalNow()) - Date.Year([入职日期])`
4. 添加「薪资等级」列：
   - 使用条件列：基本工资 < 8000 为"初级"，< 15000 为"中级"，否则"高级"

### Step 6: 清洗库存数据

**操作**：
1. 选择库存数据查询
2. 添加「周转率」计算列：
   - 公式：`[出库数量] / (([期初库存] + [期末库存]) / 2)`
3. 添加「库存预警」条件列：
   - 期末库存 < 50："库存不足"
   - 期末库存 > 300："库存积压"
   - 其他："正常"

### Step 7: 创建维度表

**产品维度表**：
1. 复制库存查询
2. 仅保留：产品ID、产品名称、类别
3. 删除重复行

**客户维度表**：
1. 复制销售查询
2. 仅保留：客户ID
3. 删除重复行
4. 添加「客户名称」列：使用 Power Query 的「添加前缀」功能，添加"客户"前缀

**日期维度表**：
1. 使用 DAX 在 Power Pivot 中创建

### Step 8: 加载到数据模型

**操作**：
1. 对每个查询选择「关闭并上载到」
2. 选择「仅创建连接」
3. 勾选「将此数据添加到数据模型」

## 阶段三：数据建模（Power Pivot）

### Step 9: 建立关系

**操作**：
1. 「Power Pivot」→「管理」→「关系图视图」
2. 建立以下关系：

| 主表 | 从表 | 关联字段 | 关系类型 |
|------|------|---------|---------|
| 产品维度 | 销售数据 | 产品ID | 一对多 |
| 客户维度 | 销售数据 | 客户ID | 一对多 |
| 日期维度 | 销售数据 | 日期 | 一对多 |

### Step 10: 创建日期维度表

**操作**：
1. 在 Power Pivot 中「设计」→「新建表」
2. 输入公式：
```dax
= CALENDAR(DATE(2024,1,1), DATE(2024,12,31))
```
3. 添加计算列：
```dax
年份 = YEAR([Date])
月份 = MONTH([Date])
月份名称 = FORMAT([Date], "M月")
季度 = "Q" & CEILING([月份]/3, 1)
```

## 阶段四：DAX 度量值

### Step 11: 财务度量值

**操作**：在 Power Pivot 公式栏中依次创建：

```dax
总收入 := CALCULATE(SUM(财务数据[金额]), 财务数据[科目类型] = "收入")
总成本 := CALCULATE(SUM(财务数据[金额]), 财务数据[科目类型] = "成本")
总利润 := [总收入] - [总成本]
利润率 := DIVIDE([总利润], [总收入], 0)
成本率 := DIVIDE([总成本], [总收入], 0)
```

### Step 12: 销售度量值

```dax
总销售额 := SUM(销售数据[销售额])
总销量 := SUM(销售数据[数量])
总订单数 := DISTINCTCOUNT(销售数据[订单ID])
平均客单价 := DIVIDE([总销售额], [总订单数], 0)
```

### Step 13: 人力资源度量值

```dax
员工总数 := COUNTROWS(人力资源)
在职员工数 := CALCULATE(COUNTROWS(人力资源), 人力资源[离职状态] = "在职")
离职人数 := CALCULATE(COUNTROWS(人力资源), 人力资源[离职状态] = "离职")
离职率 := DIVIDE([离职人数], [员工总数], 0)
平均薪资 := AVERAGE(人力资源[基本工资])
总薪资 := SUM(人力资源[基本工资])
```

### Step 14: 库存度量值

```dax
总库存金额 := SUM(库存数据[库存金额])
库存SKU数 := DISTINCTCOUNT(库存数据[产品ID])
平均周转率 := AVERAGE(库存数据[周转率])
```

### Step 15: 综合分析度量值

```dax
人均产值 := DIVIDE([总销售额], [在职员工数], 0)
人均利润 := DIVIDE([总利润], [在职员工数], 0)
薪资占收入比 := DIVIDE([总薪资]*12, [总收入], 0)
```

## 阶段八：分析结论与建议

### 关键发现

基于数据分析，以下是关键发现：

**财务方面**：
- 年度总收入：查看「综合KPI」表中的总收入
- 利润率水平：查看利润率是否在合理区间（通常 15%-30%）
- 成本结构：管理费用和研发费用占比是否合理

**销售方面**：
- 最佳销售区域：查看「区域销售」表中的第一名
- 明星产品：查看「产品销售」表中的TOP5
- 客户结构：查看「客户ABC」表中A类客户占比（理想状态：A类占80%收入）

**人力资源方面**：
- 离职率水平：健康水平通常 < 15%
- 薪资结构：各部门平均薪资是否合理
- 绩效分布：是否存在绩效两极分化

**库存方面**：
- 周转率：健康水平通常 > 2
- 滞销品：查看「库存明细」表中周转率 < 1 的产品
- 库存金额：是否占用过多流动资金

### 改进建议

1. **销售优化**：
   - 加大对最佳区域的市场投入
   - 对滞销产品进行促销或淘汰
   - 重点维护A类客户，提升B类客户价值

2. **成本控制**：
   - 分析成本率较高的月份，找出原因
   - 优化管理费用结构
   - 评估研发投资回报率

3. **人才管理**：
   - 如离职率偏高，分析原因并制定留人计划
   - 建立绩效与薪资挂钩的激励机制
   - 加强关键岗位人才储备

4. **库存优化**：
   - 对库存积压产品进行清仓处理
   - 建立库存预警机制
   - 优化采购计划，减少资金占用

## 常见问题

**Q: 数据透视表不刷新？**
A: 右键透视表 →「刷新」，或「数据」→「全部刷新」

**Q: DAX度量值显示错误？**
A: 检查公式语法，确认表名和列名正确，注意中英文标点

**Q: 切片器不联动？**
A: 选中切片器 →「切片器工具」→「报表连接」→ 勾选所有透视表

**Q: 仪表板打印格式混乱？**
A: 「页面布局」→「纸张大小」→ 调整为横向，设置打印区域
